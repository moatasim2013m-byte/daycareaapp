# Illumine App: Teacher User Guide (Operational Playbook)

This internal playbook captures the practical teacher workflows from Illumine’s official guide in a concise, task-first format for daycare/classroom operations.

## 1) Access & Login

### App access links
- **Android**: https://play.google.com/store/apps/details?id=com.illumine.app
- **iOS**: https://apps.apple.com/in/app/illumine-school/id1488933419?platform=iphone
- **Web**: https://school.illumine.app

### Login methods
1. **Email + password**
   - Select **Login to Illumine → Login with Email**.
   - Use registered email.
   - Use invitation password (or reset password if needed).

2. **Phone + OTP**
   - Select **Login to Illumine → Login with Phone number**.
   - Choose country, enter registered phone (without leading 0 / country prefix).
   - Enter OTP.

### Common login notes
- Do **not** use Sign Up if your account should already exist under a school.
- `ACCOUNT_DOES_NOT_EXIST` means account setup is pending by center admin.
- Use the exact phone number shared with the school.
- OTP issues: contact support at `info@myillumine.com`.

## 2) Student Profile Operations

The student profile is the teacher’s control center for identity, family, classroom, attendance, activities, files, notes, and safety details.

### Add student
- Home screen → **Student List** → **Add Student**.
- Required fields: Name, Gender, Classroom.
- Save and continue profile completion from Student Details.

### Update personal profile
- Open child profile → **Personal Profile**.
- Edit details (DOB, blood group, address, etc.) and save.
- In **Admission Details**, update classroom assignments and save.

### Update parents & pickup details
- Child profile → **Parents & Pickup Details**.
- Edit Father/Mother details.
- Add authorized pickup contacts with photo.
- Use **Invite Parents** to trigger parent app onboarding email.

### Additional student records
- **Payment Profile / Fee Payments**: billing and transaction visibility.
- **Emergency Contact**: primary and secondary emergency contacts.
- **Attendance tab**: check-in/check-out timeline and daily history.
- **Activities tab**: day-to-day classroom records.
- **Child’s Documents**: upload certificates/IDs/immunization docs.
- **Child Notes**: internal notes visible to staff only.

### Download allergy report (web)
Two paths:
1. **Management → Students** → menu (3 dots) → Download student allergy report.
2. **Reports → Excel Reports → Students → Allergy Report**.

## 3) Classrooms & Groups

### Manage classrooms
- More → **Classrooms** → **New Classroom**.
- Provide class name + capacity.
- Assign ownership scope (all teachers vs only self).

### Manage groups
- More → **Groups** → **+ New Group**.
- Create group and use **Assign Students**.
- Select from classroom student list (or search by name).

---

## 4) Attendance Workflows

### Check-in / check-out
- Home → Student Attendance → **Pending**.
- Select students:
  - **Mark Absent**, or
  - **Check-In** (time, classroom, temperature, late check-in flag, notify teachers).
- For checkout, open **Checked-In**, select students, then **Check-Out**.

### Move room
- Home → **Checked-In**.
- Select students → **Move Room** → choose target classroom → Done.

### Kiosk attendance
- More → **Kiosk**.
- Enter center kiosk code, then student kiosk code.
- Complete check-in details and submit.

### Attendance reports (web)
- Web app → **Reports**.
- Choose report in Student SIGN IN/OUT section (e.g., Daily Attendance).
- Set date range + classroom/student scope.
- Generate and download from Recently Generated Reports.

---

## 5) Daily Activity Reporting

### Food / nap / diaper
- Home → **Post Activity** (3-dot menu).
- **Food**: meal type, menu, quantity, media, visibility toggles.
- **Nap**: start nap and end nap flows with timestamps.
- **Potty/Diaper**: type, subtype, optional attachments.

### Learning activity / announcement / incident / medication
- **Learning activities** (e.g., Block Hour): observations + media + staff-only option.
- **Announcements**: classroom-wide or student-targeted, optional email copy.
- **Incidents**: description, action taken, notes, attachments.
- **Medication**: medicine name, dosage, optional prescription/media.

### File share & virtual class
- **File Upload** activity for documents/media sharing.
- **Virtual Class** activity for scheduling and parent invites (after Zoom setup).

---

## 6) Parent Communication & Action Queues

From **Connect** tab, teachers process inbound parent requests:
- **Notes** (food, medicine, pickup/drop, others) → Mark Done.
- **Concerns** → comment and Mark Resolved.
- **Leaves** → review/approve; moves to Reviewed.

---

## 7) Calendar, Events & Holidays

### Events
- Calendar → **Events** → New Event.
- Add title, description, date/time, classroom scope, optional parent notification and banner.

### Holidays
- Calendar → **Holidays** → New Holiday.
- Add holiday name + date range.

---

## 8) Schedule Planner (Time Table)

### Create schedule (mobile)
- More → **Schedule Planner** → Add Schedule.
- Define activity type, title/description, start/end time, assigned teachers.
- Use advanced options for recurring events.

### Bulk upload schedule (web)
- Calendar → **Add schedule using excel**.
- Download template and upload per required mode:
  - Single day
  - Specific date range
  - Multiple days (weekday-based)

---

## 9) Lessons, Assignments, Submissions

### Lesson creation
- Navigation tray → **Lessons**.
- Lessons → **New Lesson**.
- Add lesson name, category, milestones, media/video references.

### Weekly lesson plan
- Learning screen → **Weekly Lesson Plan**.
- Add lessons day-wise from lesson repository.

### Assignments
- Share lesson to selected students.
- Add optional note + due date.
- Track in **Assignments** and **Submissions** tabs.
- Teachers can comment and award stars on submissions.

### Curriculum documents
- Web: Learning → Curriculum Document → Add Document.
- Mobile: More → Document → Upload File.

---

## 10) Assessment & Portfolio (Web)

### Milestones setup
- Learning → **Milestones**.
- Add Development Area, then add milestones within it.

### Student assessment
- Learning → **Student Assessment** → select student.
- Create assessment title.
- Record progress per milestone (Developing/Advanced/Limited/etc.).
- Add notes and evidence photos.
- Save, then **Download Portfolio** or **Share Portfolio** with parents.
